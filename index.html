Guia Completo para o Projeto Custom-Script do Help Center

Visão Geral

O custom-script permite ajustes no visual e nas funcionalidades do Help Center, trazendo flexibilidade para adaptar o site às necessidades dos usuários.

Aqui você aprenderá a clonar, configurar e publicar suas modificações. Vamos explorar a estrutura do projeto e os comandos principais.

Estrutura do Projeto

  • Pasta js: Scripts de JavaScript para ajustes e novas funcionalidades.
  • Pasta do Produto: Scripts específicos para cada produto.
  • Pasta css: Arquivos de estilos visuais para o Help Center.

Requisitos

Certifique-se de ter o ambiente de desenvolvimento pronto com as seguintes ferramentas:

  • Visual Studio Code: Editor de código.
  • Node.js: Plataforma para scripts JavaScript.
  • Git: Controle de versões.

Instruções de Configuração

Siga os passos abaixo para configurar o projeto:

Clonando o Repositório:

git clone https://github.com/confluencendd/custom-scripts.git

Selecionando a Branch de Teste:

git checkout Test

Instalando Dependências:

npm install

Desenvolvimento e Teste

Use os comandos abaixo para testar localmente:

Help Center de Validação:

npm run nddvalidacao-dev

Help Center NDD Orbix:

npm run nddorbix-dev

Todos os Help Centers:

npm run all-hc-dev

Preparando para Produção

Quando estiver pronto para publicar:

Help Center de Validação:

npm run nddvalidacao-build

Help Center NDD Orbix:

npm run nddorbix-build

Problemas Comuns e Soluções

  • Cache do Help Center não atualiza: Limpe o cache do JS Deliver e do navegador.
  • Erro de Build: Remova a pasta .parcel-cache e gere novamente.
